## Grooming notes

- Start with diagnostics/read-only affordances before adding mutating repair tools.
- Preserve canonical refs in persisted state; short aliases should be presentation/resolution affordances only.
- Avoid broad compatibility shims unless data safety requires them; prefer explicit migration/doctor commands with dry-run output.
